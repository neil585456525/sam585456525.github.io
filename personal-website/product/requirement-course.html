# 課程主軸

以專題式方式授課，選定主題後，以其為主軸進行設計與原型開發。

核心素養：

  1. 可以用科學和系統化的方式收集和分析需求。
  2. 利用將理論實作的方式增強應用能力。
  3. 學會理解自身問題，並選擇相應工具的能力。

授課配置：

  1. 課程內容講解 70%
  2. 專案進度討論 30%

期末需繳交

  1. 需求研究紀錄
  2. 原型設計圖
  3. 軟體需求文件

# 課程時程進度安排

# 第一週 - 理解開發功能的源頭 - 人

  1. 從功能反推想法 > 需求 > 角色
  2. 理解需求的本質以實際案例驗證「功能、客戶想法與客戶需求的差別」
  3. 透過實際產品服務，理解和練習找出利害關係人
  4. 對利害關係人利用人物誌塑模

額外目標： 挑選組別題目

作業： 針對題目尋找利害關係人，並製作人物誌。

# 第二週 - 收集和理解需求

  1. 介紹收集需求的常用方法和注意事項（問卷、訪談）
  2. 案例研究
  3. 利用使用者故事分類和找尋未被挖掘的需求
  4. 詢問思考框架：WHY&WHY、5W1H

作業： 針對題目的利害關係人蒐集關鍵意見和潛在需求。

# 第三週 - 需求分析與解決方案

  1. 需求分類
  2. 需求優先度與價值分析
  3. 解決方案研究（相似產品研究）

作業： 將需求分類並進行分析，提出解決方案構想。

# 第四週 - 流程設計與原型製作

  1. 理解流程圖觀念
  2. 學習相關流程與原型製作工具
  3. 學習 MVP 觀念製作測試原型

作業： 製作流程圖和原型圖。

# 第五週 - 發表與反饋

  1. 發表原型和研究成果
  2. 學習製作專案需求報告書

作業： 尋找利害關係人測試原型和回饋意見。

# 第六週 - 導入軟體需求塑模與設計方法

  1. 藉由專案研究成果進行需求塑模
  2. 結合 UML 和物件導向等軟體設計方法

作業： 將需求以系統形式進行規劃

# 備註

緩衝時間一至兩週
